# SafeMe MVP stable checkpoint

Αυτό το έγγραφο καταγράφει τη σταθερή κατάσταση του SafeMe MVP και τους βασικούς κανόνες συνέχισης της ανάπτυξης μετά την πρόσφατη παλινδρόμηση στα mobile buttons και το rollback. Είναι σημείο αναφοράς για μελλοντικά PRs και δεν αποτελεί δήλωση πιστοποίησης, εγγύησης ή επίσημης έγκρισης της εφαρμογής.

## A. Τρέχουσα σταθερή κατάσταση

- Τα mobile buttons λειτουργούν.
- Η πλοήγηση λειτουργεί.
- Η προσθήκη και αποθήκευση επαφών λειτουργεί.
- Η αποθήκευση προφίλ λειτουργεί.
- Τα accordions στις Ρυθμίσεις λειτουργούν.
- Η ροή δοκιμής SOS λειτουργεί.
- Υπάρχουν δημόσιες σελίδες για Privacy, Terms και Support.
- Τα links για legal/support υπάρχουν στις Ρυθμίσεις, εφόσον έχουν ήδη προστεθεί στην τρέχουσα έκδοση.

## B. Τι θεωρείται κλειδωμένο για τώρα

- Το SOS πρέπει να λειτουργεί χωρίς login.
- Το account login παραμένει προαιρετικό.
- Ο συγχρονισμός επαφών δεν πρέπει να μπλοκάρει το SOS.
- Οι legal/support σελίδες πρέπει να παραμείνουν στατικές.
- Η εφαρμογή παραμένει μόνο στα Ελληνικά μέχρι να υλοποιηθεί σωστό σύστημα i18n.
- Δεν γίνονται global αλλαγές σε tap/click/PWA, εκτός αν είναι απολύτως απαραίτητο.

## C. Lessons from regression

- Αποφεύγουμε μεγάλες αλλαγές UI, PWA και tap/click στο ίδιο PR.
- Ελέγχουμε τα κουμπιά σε iPhone μετά από κάθε UI PR.
- Χρησιμοποιούμε rollback γρήγορα όταν αποτυγχάνουν πολλαπλά patches.
- Προχωράμε με ένα PR τη φορά και μετά κάνουμε mobile test.

## D. Manual test before continuing

Πριν συνεχιστεί η ανάπτυξη, γίνεται ο παρακάτω χειροκίνητος έλεγχος:

- Άνοιγμα της εφαρμογής σε iPhone.
- Έλεγχος πλοήγησης.
- Έλεγχος Contacts → Προσθήκη επαφής.
- Έλεγχος αποθήκευσης Profile.
- Έλεγχος Settings accordions.
- Έλεγχος Privacy/Terms/Support links.
- Έλεγχος SOS σε test mode.
- Τερματισμός SOS.

- Προσθήκη πιο καθαρής καθοδήγησης για test mode.

- Προετοιμασία screenshots για store listing.
- Μελλοντικό σύστημα i18n για Ελληνικά/Αγγλικά.
- Μελλοντικός σχεδιασμός native app και store review.
